"Operation Broken Bridge": Multi-Cloud Infrastructure Recovery

By Raghava Gatadi • October 12, 2024

Led a complex multi-cloud incident response operation involving both Google Cloud Platform (GCP) and Amazon Web Services (AWS). The incident involved unauthorized access to infrastructure resources and data exfiltration attempts.

Initial Detection

Detected unusual API activity in GCP Cloud Audit logs showing bulk downloads from Google Cloud Storage buckets. Concurrently, AWS CloudTrail logs revealed anomalous IAM role assumption patterns and increased API call volumes.

Triage & Containment

Immediately isolated affected resources on both platforms by revoking compromised credentials and implementing temporary network restrictions. Established a secure incident response environment to prevent further lateral movement.

Data Exfiltration Investigation

Used gcloud CLI to analyze artifacts from compromised Google Cloud Storage buckets, identifying potential data that was accessed or modified. Tracked data transfer patterns to determine scope of potential exfiltration.

Infrastructure Recovery

Coordinated deployment of a hybrid recovery solution using AWS CloudFormation templates for backend infrastructure and GCP e2-micro VMs for frontend services. Implemented enhanced security controls including stricter IAM policies and VPC flow logging.

Cross-Platform Analysis

Correlated logs between GCP and AWS to identify attack vectors that leveraged multi-cloud configurations. Created detailed timeline of attacker activities across both platforms, revealing coordinated efforts to establish persistence.

Lessons Learned

This incident highlighted the complexity of securing multi-cloud environments and the importance of consistent logging and monitoring across platforms. Key improvements included implementing cross-platform SIEM correlation and establishing standardized security baselines.

Key Takeaways:

  • Multi-cloud incidents require coordination across different platform-specific tools and procedures
  • Consistent logging standards across platforms are crucial for effective incident response
  • Network segmentation and zero-trust principles are essential in multi-cloud architectures
  • Regular security assessments should cover cross-platform attack vectors
