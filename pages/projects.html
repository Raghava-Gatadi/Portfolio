Detection Engineering

Splunk Enterprise | AWS CloudTrail | MITRE ATT&CK Mapping

Designed and implemented a full-cycle detection strategy using the Invictus IR Red Team dataset. I developed custom logic to distinguish between automated scanning tools and legitimate administrative activity within AWS.

  • Detection Logic Development: Wrote 6 custom SPL (Search Processing Language) rules utilizing statistical thresholding (stats dc()) and time-binning to identify the AWS Kill Chain.
  • MITRE Mapping: Aligned all detection coverage to the MITRE ATT&CK Cloud Matrix, specifically targeting Reconnaissance (TA0043) through Data Destruction (TA0040).
  • Advanced Tuning: Implemented eval severity=case() logic to grade alerts dynamically based on failed privilege escalation spikes and suspicious AssumeRole patterns.
  • High-Fidelity Rules Developed:
    • API Reconnaissance Spike: Detects automated enumeration via distinct call volume.
    • SSM Parameter Enumeration: Identifies bulk credential theft from Systems Manager.
    • Failed Privilege Escalation: Monitors AccessDenied spikes with dynamic grading.
    • CloudTrail Tampering: Immediate alerts for StopLogging or DeleteTrail events.
    • Lateral Movement: Detects "Role Chaining" and anomalous AssumeRole chains.
    • Mass Destruction: Correlates bulk resource deletion with security log disabling.
